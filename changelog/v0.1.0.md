# v0.1.0 — encrypt-then-embed + steganalysis probe

## Headline

Two new capabilities on top of v0.0.1's steganography + watermarking:

- **Encrypt-then-embed** (opt-in). `lateo hide --passphrase <PP>` seals the
  payload with chacha20poly1305 (key derived from the passphrase via
  argon2id, 7 MiB / t=3) before embedding. `lateo extract --passphrase <PP>`
  decrypts. Encrypted envelopes use a distinct magic (`LATEOSGE`) so a
  plaintext extract on an encrypted image returns a clear error asking for
  the passphrase. **The default build is unchanged and zero-cost** — the
  crypto deps (`chacha20poly1305`, `argon2`, `getrandom`) only compile
  behind `--features encryption`, satisfying the rust-oss
  zero-cost-feature invariant.

- **`lateo probe`** — steganalysis self-check. Computes per-channel
  (R, G, B) chi-square statistics (Westfeld–Pfitzmann style) over the
  `(2i, 2i+1)` value pairs, plus the LSB-equalised-pair fraction
  (pairs with `|Δ| ≤ 1`). Prints a heuristic verdict ("probably natural"
  / "likely stego (LSB equalised)"). With `--out`, also writes the
  requested bit plane (default LSB) as a black/white PNG for visual
  inspection. **Heuristic, not a proof** — small payloads in noisy covers
  stay below the radar by design.

## Compatibility

- CLI: `hide` / `extract` gain `--passphrase`; new verb
  `probe -i <img> [--plane 0..7] [--out bitplane.png]`.
- Backward compatible: existing v0.0.1 invocations unchanged.
- Tests: 26 default + 3 encrypted = 29 pass with `--features encryption`.

## Install

Prebuilt, cosign-signed binaries (Linux x86_64/arm64, macOS x86_64/arm64,
Windows x86_64) are attached to the
[v0.1.0 release](https://github.com/ljh-sh/lateo/releases/tag/v0.1.0).
To enable encrypt-then-embed, build with `--features encryption`:

```bash
cargo install --git https://github.com/ljh-sh/lateo --features encryption
```
