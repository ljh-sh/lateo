# v0.0.1 — steganography + watermarking engines

## Headline

`lateo` can now hide and recover data. Three engines behind one CLI:

- **steganography** (`hide` / `extract`) — LSB keyed-spread. A Fisher–Yates
  shuffle seeded from the key scatters a framed payload (magic + len + CRC-32)
  across the R/G/B LSBs; extraction needs the key to locate it. v1 does **not**
  encrypt the payload (the key controls *where*, not *what*) —
  encrypt-then-embed is the next step.
- **watermark, fragile** (`mark` / `verify --mode fragile`) — same keyed-spread
  core carrying an owner id. Survives lossless re-saves, breaks on any pixel
  edit: a tamper detector.
- **watermark, robust** (`mark` / `verify --mode robust`) — 8×8-block DCT
  quantisation-index modulation (step Δ = 24) on a mid-frequency luma
  coefficient. Empirically survives JPEG q70 (~99% block match) and degrades
  below q50.

The 8×8 DCT basis is computed from `std::f64::cos` (no `rustdct`); the PRNG
(SplitMix64), key hash (FNV-1a), Fisher–Yates shuffle, and CRC-32 are
hand-rolled — so the whole engine adds only `image` (png) and `clap` as
dependencies.

## Compatibility

- CLI surface: `hide` / `extract` / `mark` / `verify` (`--mode fragile|robust`)
  wired via clap; `probe` (steganalysis) is a stub.
- 18 unit tests green; `cargo fmt`, `clippy -D warnings`, `cargo test`,
  `cargo build --release` all clean; end-to-end smoke validated (steg round-trip,
  wrong-key failure, fragile tamper detection, robust JPEG-q70 survival).
- PNG only (lossless carrier). JPEG-domain embedding and `probe` are deferred.

## Install

**Prebuilt** binaries (cosign-signed, Linux x86_64/arm64, macOS x86_64/arm64,
Windows x86_64) with SHA256SUMS are attached to the
[v0.0.1 release](https://github.com/ljh-sh/lateo/releases/tag/v0.0.1). Verify
each tarball with `cosign verify-blob` (sigstore bundle) and
`sha256sum -c SHA256SUMS --ignore-missing`.

**From source**:

```bash
cargo install --git https://github.com/ljh-sh/lateo
```
